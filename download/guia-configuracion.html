3 Paso 1: Crear el Archivo .env

El archivo .env le dice al proyecto donde guardar la base de datos SQLite de trading.

1 Abrir PowerShell en la carpeta del proyecto

En PowerShell, navega a la carpeta donde descargaste el proyecto SynthTrade Pro. Usa el comando cd para entrar a la carpeta. Asegurate de estar dentro de la carpeta que contiene el archivo package.json.

PowerShell PS C:\Users\ASUS> cd Descargas\synthtrade-pro
PS C:\Users\ASUS\synthtrade-pro> dir
# Debes ver: package.json, src/, prisma/, public/, etc.
2 Crear el archivo .env

Ejecuta el siguiente comando para crear el archivo de configuracion. Este archivo contiene la ruta donde se almacenara la base de datos local SQLite que registra todas tus operaciones de trading.

PowerShell PS C:\Users\ASUS\synthtrade-pro> echo "DATABASE_URL=file:./db/custom.db" > .env
# Verifica que se creo correctamente:
PS C:\Users\ASUS\synthtrade-pro> cat .env
DATABASE_URL=file:./db/custom.db
3 Crear la carpeta db

La base de datos se guardara dentro de la carpeta db. Asegurate de crearla si no existe.

PowerShell PS C:\Users\ASUS\synthtrade-pro> mkdir db -Force
# El parametro -Force evita error si la carpeta ya existe
⚠
Importante: El archivo .env es invisible en el explorador de Windows por defecto. Eso es normal. Si necesitas editarlo, abrelo con Notepad ejecutando notepad .env en PowerShell.
4 Paso 2: Configurar la Base de Datos

Inicializa Prisma ORM para que la base de datos funcione correctamente.

1 Generar el cliente Prisma

Este comando genera las clases de TypeScript que el bot usa para comunicarse con la base de datos. Es un paso obligatorio que crea la conexion entre tu codigo y SQLite.

PowerShell PS C:\Users\ASUS\synthtrade-pro> bun run db:generate
# Salida esperada:
# ✔ Generated Prisma Client to ./node_modules/@prisma/client
2 Crear las tablas en la base de datos

Este comando lee el archivo prisma/schema.prisma y crea las tablas TradeRecord y BotSetting en tu base de datos SQLite. Ahora si deberia funcionar sin error.

PowerShell PS C:\Users\ASUS\synthtrade-pro> bun run db:push
# Salida esperada:
# ✔ The database is already in sync with the Prisma schema.
# ✔ Generated Prisma Client
✓
Si ves el mensaje verde "The database is already in sync", la base de datos esta lista. Este paso anteriormente daba error porque no existia el archivo .env.

5 Paso 3: Iniciar el Bot

Levanta el servidor de desarrollo y abre el bot en tu navegador web.

1 Iniciar el servidor de desarrollo

Ejecuta el comando de desarrollo. Esto compila la aplicacion Next.js y la pone disponible en tu red local. El proceso puede tardar entre 15 y 60 segundos la primera vez.

PowerShell PS C:\Users\ASUS\synthtrade-pro> bun run dev

# ▲ Next.js 16.x.x
# - Local: http://localhost:3000
# ✓ Ready in 3.2s
2 Abrir en el navegador

Una vez que el servidor este listo, abre tu navegador web (Chrome, Edge, Firefox) y navega a la direccion que muestra la consola. Veras la interfaz del bot SynthTrade Pro con el tema oscuro.

Navegador Abre en tu navegador:
http://localhost:3000
ⓘ
El servidor debe quedar ejecutandose. No cierres la ventana de PowerShell mientras uses el bot. Si la cierras, el servidor se detiene y no podras acceder al bot en el navegador. Para detenerlo, presiona Ctrl + C en la ventana de PowerShell.
6 Paso 4: Obtener Token de Deriv

Necesitas un token de API de Deriv para que el bot pueda operar en tu cuenta.

1 Crear o acceder a tu cuenta de Deriv

Ve a la pagina oficial de Deriv y crea una cuenta si no tienes una. Recomiendo usar una cuenta de DEMO primero para probar el bot sin arriesgar dinero real. Puedes cambiar a cuenta real cuando estes seguro del funcionamiento.

Navegador Pagina oficial de Deriv:
https://app.deriv.com

Registrate con tu email y verifica tu cuenta.
Selecciona "Cuenta Demo" para pruebas sin riesgo.
2 Generar el Token de API

El token de API es como una contrasena que le permite al bot ejecutar operaciones en tu nombre. Sigue estos pasos para generarlo desde la plataforma de Deriv.

a) Entra a https://app.deriv.com/account/api-token

b) Inicia sesion con tu cuenta de Deriv

c) En el campo "Token name" escribe: SynthTradePro

d) Selecciona los siguientes scopes (permisos):

Scope Para que sirve Necesario
Read Leer saldo, estado de cuenta y datos personales SI
Trade Ejecutar operaciones de compra/venta en tu cuenta SI
Payments Gestionar depositos y retiros NO
Admin Acceso administrativo completo NO
⚠
Solo activa Read y Trade. No actives Payments ni Admin. El bot solo necesita leer datos y ejecutar operaciones. Menos permisos = mas seguridad para tu cuenta.

3 Copiar el Token generado

Despues de crear el token, Deriv te mostrara una cadena de texto larga (aproximadamente 30 caracteres). Copia esta cadena completa. Solo se mostrara una vez por seguridad, si la pierdes tendras que generar un nuevo token.

Ejemplo de Token # El token se vera algo asi (NO uses este, es de ejemplo):
abc123def456ghi789jkl012mno345pqr
⚠
Nunca compartas tu token. Es equivalente a la contrasena de tu cuenta de trading. Cualquier persona con acceso a tu token puede operar con tu saldo.

8 Estrategias de Trading Disponibles

El bot incluye 5 estrategias que puedes combinar. Puedes activar varias al mismo tiempo y el bot usara un sistema de consenso para decidir cuando operar.

Estrategia Como Funciona Ideal Para
RSI Detecta condiciones de sobrecompra (RSI > 70) y sobreventa (RSI < 30) para anticipar reversiones del precio Volatility, Crash
MA Crossover Cruce de medias moviles (5 y 20 periodos). Compra cuando MA rapida cruza por encima, vende cuando cruza por debajo Todos los mercados
Bollinger Bands Canal de volatilidad. Opera cuando el precio toca las bandas superior o inferior, esperando un rebote Volatility, Jump
Spike Detection Detecta movimientos bruscos del precio (spikes) que superan un umbral definido. Estrategia agresiva Boom, Crash
Composite Combina las 4 estrategias anteriores. Solo opera cuando 60% o mas de las estrategias estan de acuerdo Conservadores
9 Consejos de Seguridad y Riesgo

Reglas fundamentales para operar de forma segura con el bot.

⚠
Empieza con cuenta DEMO. Nunca uses dinero real hasta que hayas probado el bot durante al menos 1-2 semanas y entiendas como se comporta cada estrategia en diferentes condiciones de mercado.
1 Configura limites de riesgo en el panel de Risk Management

El bot tiene un panel de gestion de riesgo donde puedes definir limites automaticos. Configura un limite de perdida diaria (ejemplo: $10), un objetivo de ganancia (ejemplo: $20) y un numero maximo de perdidas consecutivas antes de detener el auto-trading. Estos parametros protegen tu capital de perdidas excesivas.

2 Usa monto inicial pequeno

Configura el monto por operacion en un valor bajo como $0.35 o $1.00. Es preferible hacer muchas operaciones pequenas que pocas operaciones grandes. El sistema Martingale (que duplica la apuesta tras una perdida) puede aumentar rapidamente el riesgo si no se configura con limites estrictos.

3 Monitorea el bot regularmente

Aunque el bot opera automaticamente, es recomendable revisarlo periodicamente. Revisa el panel de metricas de rendimiento, el historial de operaciones y asegurate de que las estrategias se estan comportando como esperas. Los mercados cambian y una estrategia que funciona hoy puede no funcionar manana.
